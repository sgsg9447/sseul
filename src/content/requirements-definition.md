# 요구사항 정의서 — 국비지원 목공 직업훈련기관 웹사이트 리뉴얼

| 항목 | 내용 |
| --- | --- |
| 문서 | 요구사항 정의서 (Requirements Definition) |
| 프로젝트 | 국비지원 목공 직업훈련기관 웹사이트 리뉴얼 |
| 대상 | 실제 운영 중인 직업훈련기관 (사명 비공개) |
| 역할 | 기획 · 설계 · 디자인 · 개발 전 과정 단독 진행 |
| 선행 문서 | 서비스 기획서 (문제·사용자·목표·범위) |
| 우선순위 표기 | **M**ust / **S**hould / **C**ould / **W**on't (MoSCoW) |
| 버전 | v1.0 |

> **작성 기준:** 본 문서는 서비스 기획서의 근본 원인(A·B·C·D)과 설계 원칙에서 도출한 요구사항을, 기능·데이터·비기능·정책으로 **구조화하고 추적 가능하게** 정리한 것이다. 완성된 리뉴얼을 요구사항 관점에서 재정의한 문서이며, 각 요구사항은 *어떤 문제에서 나왔는지(도출 근거)와* *어느 화면에서 충족되는지(관련 화면)를* 함께 명시한다.

---

## 1. 요구사항 도출 체계

요구사항은 직감이 아니라 **문제 → 원인 → 설계 원칙 → 요구사항**의 사슬에서 나온다.

```
20개 증상 ─수렴→ 4개 근본 원인(A·B·C·D) ─뒤집기→ 설계 원칙 ─구체화→ 요구사항군
A 콘텐츠가 데이터 아님   → 데이터 모델링 요구사항 (FR-CRS, FR-DAT)
B 행동·전환 흐름 미설계  → 전환 흐름·피드백 요구사항 (FR-APP, FR-INQ)
C 게시판 전용           → 목적별 전용 기능 요구사항 (FR-INQ, FR-APP)
D 정보 위계 실패        → 신뢰 신호·정보 위계 요구사항 (FR-TRU)
```

요구사항군 약어: **CRS**(과정) · **APP**(신청) · **INQ**(문의) · **SUP**(국비지원) · **GAL**(훈련사진) · **TRU**(신뢰 신호) · **ADM**(관리자) · **DAT**(데이터) · **NFR**(비기능) · **POL**(정책)

## 3. 데이터 요구사항

콘텐츠를 데이터로 모델링한다는 메타 원칙①의 실체. 사용자 화면은 '읽고', 관리자 화면은 '쓰며', **같은 데이터를 본다** — 수기 신청서·이미지 수정이 사라지고 사이트가 업무 도구가 되는 지점.

| 엔티티 | 핵심 속성 | 비고 |
| --- | --- | --- |
| **Course** | name · type · tags · 회차/시간 · **fundingType** · 커리큘럼(NCS) · 모집안내 · recruitStatus | fundingType이 신청 분기를 구동 |
| **Schedule** | 과정별 일정 | 과정에 종속 |
| **Application** | 이름 · 연락처 · 관심 과정 · 상태(접수→상담→등록확인) | 비로그인 생성 |
| **Inquiry** | 이름 · 연락처 · category · 내용 · 상태(답변대기→답변완료) | 비로그인 생성 |
| **Post** | category(훈련사진/수강일지/수료식) · 이미지 | 갤러리 |
| **History** | 연도별 인증·수상 | 학원소개 |

> 핵심: 과정 목록·상세는 매년 흔들려도 **`fundingType` → 신청 흐름의 매핑은 안정적**이다. 그래서 과정별로 분기 로직을 만들지 않고 유형(A·B·C)으로 분기한다.

## 7. 요구사항 추적 매트릭스

근본 원인이 요구사항·화면까지 누락 없이 이어졌는지 검증하는 표.

| 근본 원인 | 대표 요구사항 | 관련 화면 |
| --- | --- | --- |
| **A. 콘텐츠가 데이터 아님** | FR-CRS-01·03·04, FR-SUP-01, 데이터 모델 | COURSE-LIST, COURSE-DETAIL, SUPPORT |
| **B. 행동·전환 흐름 미설계** | FR-APP-02·04, FR-INQ-03, FR-TRU-01 | HOME, APPLY, INQUIRY |
| **C. 게시판 전용** | FR-INQ-01·02, FR-APP-01 | INQUIRY, APPLY |
| **D. 정보 위계 실패** | FR-TRU-01·03, FR-CRS-02 | HOME, ABOUT, GALLERY |
| **목표② 운영 도구화** | FR-ADM-01·03·04, 공유 데이터 레이어 | ADMIN |

> 20개 증상 → 4개 원인 → 요구사항 → 화면이 한 줄로 이어진다. 추적 매트릭스가 채워진다는 것은 **누락 없이 설계됐다**는 뜻이다.
