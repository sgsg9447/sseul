# Hoonjo 포트폴리오 — 작업 기록 & 결정 로그

> 조영훈(Hoonjo) 시니어 프론트엔드 포트폴리오. 라우트 `/d/hoonjo-b9e634` (숨김 샘플·noindex).
> 기반: `hoonjo-design` 디자인 시스템 + 그가 직접 쓴 옵시디언 볼트/이력서/프로젝트 회고.

이 문서는 **무엇을 만들었고, 왜 그렇게 판단했는지**를 남긴 결정 로그다.

## 관통하는 5가지 원칙

1. **진짜 숫자만.** 디자인 시스템 예시에 박혀 있던 그럴듯한 가짜 숫자(`@hoonjo/typeset`, `hello@hoonjo.dev` 등)는 전부 버리고, 그가 실제로 측정한 값만 썼다. 검증 안 되는 숫자는 시니어 면접에서 독이다.
2. **무자비한 큐레이션.** 13개 프로젝트를 다 넣으면 그게 기존 사이트가 망한 이유(블로그 더미)다. 6개만 골랐다.
3. **정직함.** 안 만든 것(velto)·못 푼 한계(표 분할, reparenting 끊김)는 숨기지 않고 그대로 적었다. "못 푼 걸 안 푼 척하지 않는 게 라이브러리 쓰는 사람한테 정직한 것."
4. **도메인 몰라도 읽히게.** 타깃은 "교육/보안 도메인을 모르는 **프론트엔드 엔지니어**". 기술 깊이는 유지하되 도메인 포장지(조판·지문·보안관제·관제사)만 투명하게 글로싱.
5. **자급식 폴더.** 이 폴더 하나가 통째로 떨어져 나가게(추출 계약). 외부 import 0, `.hoonjo`로 스코프, lazy 청크.

## 주요 결정

### 1. 메트릭 진실성 — "Numbers do the bragging"을 진짜 숫자로
브랜드 룰이 "숫자로 증명"이라, 볼트의 실측값으로 채웠다:
- column-pager: 100장 재배치 **55–66ms**, 편집 1건 **11–16ms**, 결정적 테스트 **49개**
- 대용량 PDF: 첫 조작까지 **639,000ms → 1,310ms** (약 488배), 페이지당 2,132ms → 1,527ms
- 법무부 포털: 컬럼 정의 1벌 → **59개 화면**, 696/1,299 커밋
- 타일맵 268MB → 10.7MB (단, 해당 프로젝트는 큐레이션에서 빠져서 히어로에서도 제거)

> ⚠️ `639,000ms → 1,310ms`는 처음엔 볼트에 근거가 없어 뺐다가, 사용자가 풀 회고를 주면서 출처가 확인돼 메인으로 복귀시켰다.

### 2. 큐레이션 기준과 선정
**기준:** ① 범위 > 중복 (같은 역량 증명은 하나만) ② 임팩트/측정값 ③ 시니어 시그널(트레이드오프·정직한 한계·오너십) ④ 최신성

**선정 6 (역량 축이 안 겹치게):**
| 작업 | 증명 역량 | 회사 |
|---|---|---|
| column-pager (flagship) | 렌더링/측정 + OSS | @Bookips |
| 대용량 PDF 10분→1초 | 성능 | @Sling |
| 59화면 포털 | 시스템 설계/추상화 | @Zipida·법무부 |
| 디자인 시스템 자동화 | 라이브러리/DX | @Bookips |
| ML 학습 마법사 | 복잡한 상태/풀스택 | @Zipida·KISTI |
| 블랙홀 | 그래픽스 (라이브) | 개인 |

**보류(약해서가 아니라 축이 겹쳐서):** 본문 에디터, prod/local PDF, CTF 채점, BFF, POD, 엑셀 파이프라인, combineLifecycles, corocoro(6년 전·본인도 "어린 결정"), velto(미구현), gatsby. → 언제든 교체 가능.

### 3. Flagship = 케이스 스터디
column-pager는 압축 카드가 아니라 **풀 케이스 스터디**로 짰다. 문제(2년 환불) → **다섯 번의 시도**(각 막힌 점) → 핵심 인사이트("실패를 버리지 않고 각도만 바꿔 합치다") → v1→v2→v3 진화 + 정직한 한계 → 결과. 시니어 면접관이 제일 보는 "어려운 문제를 끝까지 끌고 간 서사".
- 보내준 페이지네이션 출력 이미지는 채팅 파일을 못 꺼내서 **컴포넌트로 재현**([PagedDoc](./PagedDoc.tsx)) — 벡터라 안 깨지고 반응형.

### 4. 라이브 WebGL 블랙홀 (가짜 → 진짜)
원래 패널은 CSS `<div>` 정지 그림이었다. 그래픽 프로젝트를 정지 그림으로 보여주는 건 거짓에 가까워서, 그의 실제 기법(Canvas 별판 → WebGL 텍스처 → 셰이더 UV 왜곡)대로 **실시간 렌더**로 교체([BlackHole](./BlackHole.tsx)).
- 헤드리스 프리뷰는 WebGL 캡처를 못 해 흰 화면 → `readPixels`로 검증(9.8만 px·파란 고리 2,410px).
- **버그:** React StrictMode가 effect를 두 번 도는데, cleanup의 `loseContext()`가 컨텍스트를 죽여 리마운트 시 검은 화면. → cleanup에서 loseContext 제거, 리소스만 해제.
- 데모 배포 URL이 없어 iframe은 못 박고, 대신 실제 레포(`H8njo/webgl-black-hole`) 링크.

### 5. 회사 vs 서비스 — `@회사` 컨벤션
`Bookips · Solvook`이 뭐가 회사/서비스인지 헷갈려서 **`@회사`**로 통일(@Bookips/솔북, @Sling/ORZO, @Zipida·법무부). `@`=소속, 제품·고객은 글에서 풀어 씀.

### 6. Expertise = 역량 → 증거 지도
기술 나열이라 죽은 섹션이던 걸, **역량 → 그걸 증명한 작업**으로 바꿨다. "증명한 작업"의 프로젝트명은 **클릭하면 해당 작업으로 스크롤**(앵커 `work-*` + `scroll-margin-top`).

### 7. 콜드 리더 패스
도메인 모르는 FE가 처음부터 읽어도 막힘 없게: 떠다니던 히어로 메트릭(없는 프로젝트) 교체, 도메인어 인라인 글로싱(조판→2단 레이아웃, 지문→본문, 보안관제→관리자 포털, CRUD→추가·수정·삭제), 추상적 리드 구체화.
